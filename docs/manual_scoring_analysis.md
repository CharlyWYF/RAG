# 人工评分结果分析

基于自动化评测结果，本项目进一步采用“评分建议初稿 + 人工复核”的方式，对 41 道测试题进行了回答质量评估。评分维度包括正确性（Correctness）、完整性（Completeness）、忠实性（Faithfulness）以及检索相关性（Retrieval Relevance），每项采用 0~2 分的三档评分。通过该方式，可以在参考主流 RAG 评估思想的同时，结合网络协议问答场景的特点，对系统表现进行更细粒度分析。

从整体结果来看，系统在正确性、忠实性和检索相关性三个维度上表现较好。41 道题的平均正确性为 **1.878**，平均忠实性同样为 **1.878**，平均检索相关性达到 **1.927**。这表明，系统在大多数情况下能够检索到与问题高度相关的文档，并基于检索到的上下文给出较为可靠的回答。相比之下，平均完整性为 **1.634**，明显低于正确性与忠实性，说明当前系统的主要短板并不在“答错”，而在于“虽然答对，但展开不够充分，或对关键要点覆盖不完全”。这一结果与前期自动化评测的观察一致：系统更容易做到“方向正确”，但在某些对比题、机制题和综合题上，回答仍不够完整。

从协议类别角度看，系统在多数协议组上的表现较稳定。TLS、TCP、ICMP、ARP、HTTP/IP 等类别在四个评分维度上普遍接近满分，其中 TLS 类问题的平均正确性和忠实性均达到 **2.0**，平均完整性也达到 **1.833**，说明系统在 TLS 相关 RFC 上的回答质量相对成熟。TCP 类问题同样表现突出，平均正确性、完整性、忠实性和检索相关性均达到 **2.0**。相比之下，DNS 类问题的平均正确性为 **1.714**，平均完整性为 **1.429**，平均忠实性为 **1.714**，明显弱于 TLS 和 TCP。这说明 DNS 相关问题，尤其涉及 DNSSEC、negative caching 或协议细节区分时，系统虽然通常能检索到正确文档，但在组织回答和完整覆盖关键点方面仍有不足。UDP 类问题的检索相关性仅为 **1.0**，也提示这类问题在目标文档命中与来源相关性方面存在进一步优化空间。

从题型角度看，不同问题类型之间也呈现出较明显差异。定义类问题的平均正确性为 **1.909**，平均完整性为 **1.818**，总体表现较好；机制类问题的平均正确性达到 **2.0**，平均完整性为 **1.857**，说明系统对协议机制解释类问题具有较好的处理能力。相对而言，对比类问题的平均正确性为 **2.0**，但平均完整性仅为 **1.143**，这一现象非常值得注意。它说明系统在对比类问题上往往能够把握大方向，但容易遗漏部分差异点，因此显得“基本答对，但不够完整”。综合类问题也表现出类似趋势：正确性较高，但完整性偏低。这进一步支持了本项目当前的一个核心结论，即系统更擅长回答定义性和单机制问题，而在需要同时整合多个要点的问题上，完整性仍需加强。

证据不足类问题的评分结果也具有代表性。该类问题的平均正确性、完整性和忠实性均为 **1.5**，检索相关性为 **1.833**。这一结果说明，系统在“应保守回答”场景下表现并不完全稳定：一方面，它并未出现大规模完全错误回答，说明仍有一定边界意识；另一方面，其保守回答和拒答控制尚未完全达到理想状态，仍存在部分问题未能及时保守处理或回答边界控制不够严格的现象。这与自动化评测中保守回答成功率仅为 **50%** 的结果相呼应，表明证据不足场景下的稳健性仍然是后续需要重点优化的方向。

私有协议 SUT 的评分结果也具有一定参考意义。SUT 类问题的平均正确性为 **1.833**，平均完整性为 **1.667**，平均忠实性为 **1.833**，检索相关性为 **2.0**。由于 SUT 的知识源来自单一私有协议文档，这一结果说明系统在私有知识源接入场景下具备较好的适应能力，能够稳定命中对应文档并给出较可靠回答。不过，完整性仍略低于满分，说明即使在私有协议文档场景下，系统同样存在“答得对但还不够充分”的问题。

综合自动化评测与人工评分结果，可以得到较为一致的结论：当前系统已经具备较好的检索能力和较高的回答正确性，能够在 RFC 标准文档和私有协议文档场景下稳定工作；但其主要不足体现在回答完整性以及证据不足场景下的保守控制能力上。换言之，系统当前最需要继续优化的方向，并不是“能否找到正确文档”或“是否完全答错”，而是“如何在命中文档的前提下更完整地组织协议要点，以及如何在证据不足时更稳定地收敛回答边界”。这一结论可作为后续优化查询改写、上下文组织方式与拒答策略的重要依据。
