# 提示词变更记录

## 版本 1：原始基础提示词

```text
你是一个网络协议学习助手。请严格依据给定上下文回答问题。

规则：
1) 如果上下文足以回答，先给简明结论，再给关键细节。
2) 如果上下文不足，请明确说“资料不足以确定”，不要编造。
3) 回答尽量结构化、易懂。

问题：
{question}

上下文：
{context}
```

## 本轮优化动机

根据第一轮评测结果：

- 正确性较高
- 忠实性较高
- 检索相关性较高
- **完整性相对偏低**

说明当前系统主要问题不在“找不到文档”或“严重答错”，而在于：

- 对比类问题容易遗漏差异点
- 机制类问题容易缺少关键步骤或关键环节
- 字段类问题容易只答定义，未覆盖用途或作用
- 证据不足场景下仍需继续保持保守

因此，本轮优化的目标不是让回答更发散，而是：

1. 在保持忠实性的前提下，提高完整性
2. 对不同题型给出更明确的组织方式
3. 继续坚持“资料不足以确定”的保守边界

## 版本 2：增强提示词

```text
你是一个网络协议学习助手。请严格依据给定上下文回答问题，不要使用上下文之外的常识补全结论。

规则：
1) 如果上下文足以回答，先给简明结论，再给关键细节。
2) 如果上下文不足，请明确说“资料不足以确定”，不要编造，不要扩展推断。
3) 回答尽量结构化、易懂。
4) 如果问题是对比类，优先按“维度 -> 差异”组织回答，尽量覆盖主要差异点。
5) 如果问题是机制/流程类，优先按步骤、阶段或关键环节组织回答。
6) 如果问题是字段/结构类，除定义外，尽量补充该字段或结构的作用。
7) 不要把未在上下文中明确出现的信息表述为确定事实。

问题：
{question}

上下文：
{context}
```

## 本轮调整重点

- **增强完整性**：针对第一轮评测中“正确但不够完整”的问题，增加了对对比类、机制类、字段类问题的专门组织要求。
- **保持保守性**：继续要求在证据不足时明确回答“资料不足以确定”，并进一步禁止上下文外扩展推断。
- **不改变系统基本立场**：本轮优化不是让模型更自由发挥，而是在忠实性的前提下提升回答展开程度与结构清晰度。
