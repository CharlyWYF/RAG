# 论文实验小结（首版）

本轮实验基于当前项目构建的 RFC-first 协议知识库与私有演示协议语料，使用结构化测试问题集共 41 题进行自动化评测。其中，标准 RFC 问题 30 题，保守回答测试题 5 题，私有协议 SUT 演示题 6 题。评测结果由自动化脚本批量生成，并记录了回答内容、检索来源、目标文档命中情况以及多阶段耗时数据。

从整体结果来看，系统对目标协议文档的检索命中率较高。本轮评测共 41 题，其中 39 题命中了预期目标文档，目标文档命中率达到 **95.12%**。这一结果说明，当前知识库构建、检索链路以及查询改写机制能够较稳定地将问题映射到对应协议语料，尤其在 DNS、HTTP、TLS、TCP、IP 及 SUT 私有协议等类别上，命中率整体表现较好。

在运行性能方面，本轮实验的平均总耗时为 **8.709 秒**，平均首字响应时间为 **5.777 秒**。进一步分解各阶段耗时可以发现，查询改写平均耗时为 **1.313 秒**，检索阶段平均耗时为 **2.888 秒**，回答生成阶段平均耗时为 **4.460 秒**。从结构上看，回答生成与检索阶段仍然是当前系统时延的主要组成部分，其中回答生成阶段占比较高，说明大模型生成过程仍是总体响应时间的重要来源。另一方面，平均唯一来源数为 **2.54**，说明系统在多数问题上能够检索到不止一个来源片段，这为回答的完整性提供了一定支撑。

从不同协议类别的结果来看，系统在大多数协议组上表现稳定。DNS、HTTP、TLS、TLS/HTTP、TCP、TCP/UDP、IP、ICMP、ARP、HTTP/TLS、HTTP/IP 以及 SUT 演示协议等类别的目标文档命中率均达到 **100%**。其中，SUT 私有协议的平均唯一来源数为 **1.0**，这与其知识来源仅依赖单一私有文档的设定一致，也说明系统能够较准确地定位到私有协议文本。相比之下，UDP 类问题的命中率为 **50%**，综合类问题命中率为 **0%**，这提示我们：跨协议对比问题与覆盖面较宽的问题仍然对检索与查询改写提出更高要求，后续仍有优化空间。

在保守回答能力方面，本轮实验共设计 6 道应当保守回答的问题，系统实际触发保守回答 3 次，保守回答成功率为 **50%**。这一结果表明，当前系统已经具备一定的拒答 / 保守回答能力，但在“证据不足时避免过度推断”这一点上仍不够稳定。尤其在一些涉及真实部署性能、未来趋势或绝对化安全结论的问题上，系统仍可能给出超出 RFC 证据边界的回答。因此，后续可考虑继续加强提示词约束，或者在评测脚本中增加针对拒答表现的专项分析。

综合来看，本轮实验表明：当前系统已经具备较好的协议知识检索能力与较高的目标文档命中率，在 RFC 标准文档问答场景下表现稳定，同时也能支持私有协议知识源的接入与回答。与此同时，实验结果也暴露出两类仍需继续优化的问题：一是跨协议综合问题与少数协议类别的检索稳定性仍有提升空间；二是保守回答能力尚未完全达到理想水平。整体而言，这一结果可以支撑论文中“系统已具备较完整协议问答能力，但在复杂问题与拒答控制方面仍需持续优化”的实验结论。
