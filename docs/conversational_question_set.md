# 查询改写补充实验：自然问法测试子集

## 1. 目的

该测试子集用于补充验证：

> 查询改写是否在“口语化、非标准术语、真实用户表达风格”的问题场景下更有价值。

与主测试集相比，本子集不强调专业化和标准化提问，而是尽量模拟普通用户在没有严格协议术语意识时的真实提问方式。

因此，这组问题不适合作为主实验主体，而更适合作为：

- 查询改写补充实验
- 用户自然问法场景验证
- 论文中对查询改写适用边界的补充说明

---

## 2. 使用建议

### 2.1 推荐对比组

建议仅对比两组：

- **without_rewrite**：不启用查询改写
- **with_rewrite**：启用当前查询改写方案

### 2.2 推荐观察指标

- 目标文档命中率
- 平均唯一来源数
- 正确性
- 完整性
- 忠实性
- 检索相关性
- 平均总耗时
- 平均首字响应时间

### 2.3 推荐结论方向

该子集不一定追求“整体分数很高”，更重要的是观察：

- 查询改写是否能把模糊问题改成更适合检索的表达
- 查询改写是否比专业化问题集上更有收益
- 查询改写是否在口语化场景下更能体现价值

## 4. 推荐实验定位

本子集建议作为论文中的**补充实验**，而不是主实验主体。

推荐写法：

- 主实验：使用第二版专业化测试集评估系统整体表现
- 补充实验：使用自然问法测试子集评估查询改写在更真实用户表达场景下的价值

这样可以更自然地解释：

- 为什么查询改写在主测试集上的收益有限
- 为什么它在真实、口语化、模糊问法场景下可能更有帮助
